### Wire / decode gotchas

- Each net field is ONE handle even when it is a static array (type shown as `T[N]`).
  For a static array, the wire order is: `SerializeInt(handle, maxHandle)` then
  `SerializeInt(elemIndex, ArrayDim=N)` then the element value. (maxHandle counts the
  property once, NOT once per element - verified: handle totals match the known maxHandles.)
- Scalar (non-array, N=1) fields: handle then value directly, no index.
- Value encodings: byte=8b, bool=1b, int/float=32b, string=int32 len + chars,
  name=SerializeName, struct=member-by-member, obj ref=1 selector bit +
  SerializeInt(idx, 0x80000000 static / NetIndexCount dynamic).
- `struct Vector`/`Rotator` replicate as compressed vectors/rotators, not 3x raw float.
- GameReplicationInfo has NO replicated `Teams`/`PRIArray` net property: clients rebuild
  those arrays from the individually-replicated ROTeamInfo and (RO)PlayerReplicationInfo
  actors. Team-button data lives in ROTeamInfo net props (TeamIndex, etc.), not in GRI.
